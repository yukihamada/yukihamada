## サービス別レポート

### 1. chatweb.ai（AIエージェントプラットフォーム）

**概要**: 14チャネル対応・マルチモデルAIエージェント

**数値**:
- 訪問者: 5,200人（前月比 +12%）
- 登録ユーザー: 1,850人（前月比 +8%）
- アクティブユーザー: 820人
- MRR: 48万円（前月比 +5%）

**トピックス**:
- **新機能**: Explore Mode（複数AIモデル並列実行）リリース
- **技術**: Local LLM Fallback（Qwen3-0.6B）実装
- **プロモーション**: Product Hunt投稿 → 320 upvotes

**課題**:
- クレジット消費が早すぎる（Free Plan 100クレジット → 1日で使い切るユーザー多数）
- チャネル別プロンプトの最適化が必要

**学び**:
- Explore Modeは思ったより使われてる（DAU 120人）
- LINE Botからの流入が安定している（月400人）

## 失敗談

### 失敗1: news.xyzのSEOを放置

2月1日時点で、Google Search Consoleを見たら、インデックス数が**12ページのみ**。

180本記事書いてるのに...。

**原因**: sitemap.xmlを生成していなかった。

**対策**: Next.js の `next-sitemap` 導入 → 2日後、インデックス数120に増加。

**学び**: 基本を疎かにしない。

---

### 失敗2: chatweb.aiのクレジット計算バグ

「無料で使えすぎる」という報告が来て調べたら、クレジット計算で**床関数**使ってた。

```rust
// 間違い
let credits = (tokens / 1000.0).floor() as i64;

// 正解
let credits = (tokens as f64 / 1000.0).ceil() as i64;
```

0.8トークンも1トークンも、**1クレジット**消費すべきなのに、切り捨てで0クレジットになってた。

→ 過去ユーザーのクレジット再計算 → 一部ユーザーに返金

**学び**: 課金システムのテストは厳密に。

---

### 失敗3: jitsuflowのサポート対応漏れ

道場オーナーから「問い合わせしたのに返信ない」とクレーム。

調べたら、**Gmailの迷惑メールフォルダ**に入ってた。

→ 2週間気づかず

**対策**: サポートメールをZendeskに移行（自動振り分け）

**学び**: 手動運用は限界がある。

## 学んだこと

### 1. 小さく始めて、フィードバックを聞く

すべてのサービスが、「MVP（最小機能製品）→ フィードバック → 改善」のサイクルで成長。

最初から完璧を目指さない。

### 2. 収益化は早めに考える

Elio、webllm.app は「面白い」けど、収益ゼロ。

趣味ならOK。ビジネスなら、早めにマネタイズ方法を考える。

### 3. 自動化できるものは自動化する

- ブログ記事のSNS投稿 → Zapier
- サポート対応 → Zendesk + AI
- 月次レポート → スクリプト生成

人間がやるべきは、**創造的な部分だけ**。

### 4. 失敗は公開する

「845万円課金した話」が一番バズった。

失敗談は、共感を呼ぶ。

### 5. 健康第一

睡眠5.2時間は少なすぎる。来月は6時間目指す。
